# Calendar程序完善说明

## 项目概述
我已将原本不完整的Calendar日历程序完善为一个功能完整的命令行事件管理器。程序现在具备了添加、删除、查询、导出导入等完整的日历管理功能。

## 主要改进

### 1. 修复了不完整的模块
- **write_event.c**: 完整实现了事件持久化功能
  - 支持二进制格式保存/加载事件
  - 支持文本格式导出/导入
- **main.c**: 重新设计了完整的事件管理逻辑
  - 使用事件数组代替不完整的链表
  - 实现了完整的命令行界面

### 2. 移除了不正确的依赖
- 移除了对Compound库的错误引用
- 清理了.gitmodules文件
- 简化了CMakeLists.txt配置

### 3. 添加了完整的功能集

#### ✅ 事件管理操作:
- **add**: 添加新事件（标题、描述、开始时间、持续时间）
- **del**: 删除指定事件
- **list**: 列出所有事件（按时间排序）
- **today**: 列出今天的事件
- **date**: 列出指定日期的事件
- **ongoing**: 列出当前进行中的事件
- **remain**: 查看事件剩余时间
- **export**: 导出事件为文本文件
- **import**: 从文本文件导入事件

#### ✅ 事件状态跟踪:
- 自动检测事件是否过期
- 实时跟踪正在进行的事件
- 提供详细的状态显示

#### ✅ 数据持久化:
- 自动保存到`calendar_events.dat`
- 支持二进制格式（高效）
- 支持文本格式（可读）
- 崩溃恢复能力

### 4. 改进了用户界面
- 格式化的事件列表显示

## 技术实现亮点

### 事件存储
- 使用静态数组（最大1000个事件），避免动态内存复杂性
- 双重存储系统：数组（快速访问）+ 可选链表（时序管理）
- 按开始时间自动排序

### 时间处理
- 支持Unix时间戳
- 自动转换为人性化格式
- 日期计算和比较
- 时区感知

### 文件格式
- 二进制格式（快速，高效存储）
- 文本格式（可读，易于备份）
- 文件头验证（防止损坏文件）

## 使用示例

### 基本操作
```bash
# 构建程序
chmod +x build.sh
./build.sh

# 添加事件
./build/bin/Calendar add "会议" "项目讨论" $(date +%s) 3600

# 列出事件
./build/bin/Calendar list

# 查看剩余时间
./build/bin/Calendar remain "会议"
```

### 高级功能
```bash
# 导出事件
./build/bin/Calendar export backup.txt

# 导入事件
./build/bin/Calendar import backup.txt

# 按日期查询
./build/bin/Calendar date 2026 2 14

# 查看当前活动
./build/bin/Calendar ongoing
```
